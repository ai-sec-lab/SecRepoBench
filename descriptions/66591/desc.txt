// The function `SDP_ParseAttribute` processes a given SDP attribute line. It extracts the attribute
// type from the line and matches it against known SDP attributes such as "cat", "keywds", "tool", 
// "ptime", "recvonly", "sendonly", "sendrecv", "orient", "type", "charset", "sdplang", "lang",
// "framerate", "quality", "rtpmap", and "fmtp". Each attribute is parsed and relevant fields of the 
// `GF_SDPInfo` or `GF_SDPMedia` structures are updated accordingly. 
// 
// - For "cat", "keywds", "tool", "ptime", "type", "charset", "sdplang", "lang", "framerate", "quality", 
//   "rtpmap", and "fmtp", the function extracts values from the buffer and assigns or appends them
//   to the corresponding fields in the `sessionDescriptionInfo` or `media` structures. 
// 
// - The "recvonly", "sendonly", and "sendrecv" attributes update the send/receive capability flags.
// 
// - The "orient" attribute is assigned to media with a type check.
// 
// - The "rtpmap" and "fmtp" attributes involve more complex parsing and create new list entries 
//   for RTP mapping and FMTP attributes, adding them to the media structure's lists.
// 
// If the attribute is not recognized or is application-specific, it is stored in the attribute list
// of the `sessionDescriptionInfo` or `media` to preserve the information for later use.
