// This part processes Bluetooth Low Energy (BTLE) data packets based on the LLID (Logical Link Identifier).
// It handles different packet types: continuation fragments, start of L2CAP messages, and control PDUs.
// For continuation fragments (LLID = 0x01), it checks for missing fragments, manages L2CAP reassembly,
// and updates protocol columns and trees with fragment information. For complete L2CAP messages (LLID = 0x02),
// it manages message reassembly, updates the column info, and calls the L2CAP dissector.
// Control PDUs (LLID = 0x03) are dissected based on specific control opcodes, updating columns and protocol trees
// with control information. It also handles unknown data cases, updating the protocol tree with expert info.
